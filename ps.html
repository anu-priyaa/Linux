"ps" command in LINUX

The ps (short for process status) command is used to display information about the currently running processes on a Linux system. It provides details such as process IDs (PIDs), CPU and memory usage, and the status of processes.

Syntax:

ps [options]

where:

  • [options]: Flags to customize the output and control which processes are displayed.

Common options:

  1. Basic options:
    • ps: Displays basic information about processes started by the current user in the current terminal.
    • ps -e: Displays all processes running on the system.
    • ps -f: Provides a full-format listing with additional details.
    • ps -u username: Displays processes owned by a specific user.
  2. Extended options:
    • ps aux: Shows all processes, including those of other users and system processes.
    • ps -l: Displays detailed information in a long format.
    • ps -o : Customizes the output by specifying which columns to display.
  3. Interactive process monitoring:
    • ps -eH: Shows all processes in a hierarchical (tree) format.

    Purpose:

    • To monitor and manage processes on a Linux system.
    • To view the status of specific processes.
    • To identify resource-intensive processes for troubleshooting.
    • To gather information for system performance analysis and debugging.

    Applications:

    Some applications of "ps" command are:

    1. Monitoring System Performance:
      • Identify processes consuming high CPU or memory
    2. Troubleshooting and Debugging:
      • Check the status of specific processes during software development or system troubleshooting
    3. Identifying Zombie Processes:
      • Detect and investigate zombie (defunct) processes
    4. Kill or Manage Processes:
      • Use ps to find the PID of a process to terminate it
    5. System Administration:
      • Monitor processes by specific users or services
