"chmod" command in LINUX

The chmod (short for change mode) command is used to modify the permissions of files and directories in Linux. It controls who can read, write, or execute a file or directory.

Syntax:

chmod [options] mode file

where:

  • [options]: Optional flags to modify the behavior of the chmod command. Common options include:
    • -R: Apply changes recursively to all files and subdirectories within a directory.
    • -v: Verbose mode; displays a message for each file processed.
    • -c: Like -v, but only displays changes made.
  • mode: Specifies the new permission settings for the file or directory. It can be written in:
    • Numeric (octal) format: A three- or four-digit number representing permissions. Example: 755
    • Symbolic format: A combination of symbols to add, remove, or set permissions. Example: u+rwx, g-w, o+x
  • file: The name of the file or directory for which permissions are being changed. You can specify a single file or multiple files.

Purpose:

  • To manage file access permissions.
  • To restrict or grant file access to users, groups, or others.
  • To ensure security by limiting unauthorized access.
